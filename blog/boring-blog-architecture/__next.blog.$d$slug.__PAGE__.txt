8:Tce6,<p>I recently added a blog to this site. Rather than reaching for a CMS, I decided to build it directly into the existing Next.js application using standard tools.</p>
<h2>Requirements</h2>
<p>I wanted a place to share technical thoughts without adding another system to maintain. The requirements were simple:</p>
<ol>
<li><strong>Write in Markdown</strong>: I want to write posts in VS Code.</li>
<li><strong>Zero Runtime Cost</strong>: The blog should be statically generated.</li>
<li><strong>Fast</strong>: It should feel instant.</li>
</ol>
<h2>Markdown files and static routes</h2>
<p>I created a simple utility, <code>blogApi.ts</code>, that reads directly from the file system. It uses <code>gray-matter</code> to parse frontmatter and a <code>remark</code>/<code>rehype</code> pipeline, including <code>rehype-pretty-code</code>, to process the content.</p>
<p>Next.js makes it straightforward to turn a folder of markdown files into routes. I used <code>generateStaticParams</code> to tell Next.js which paths to build at compile time:</p>
<pre tabindex="0"><code><span><span style="color:#F97583">export</span><span style="color:#F97583"> async</span><span style="color:#F97583"> function</span><span style="color:#B392F0"> generateStaticParams</span><span style="color:#E1E4E8">() {</span></span>

<p>This ensures that <code>alexleung.ca/blog/boring-blog-architecture</code> is just a static HTML file at deploy time, not a dynamic request.</p>
<h2>The details that made it usable</h2>
<p>I also spent time on a few details that made the whole thing feel finished:</p>
<ul>
<li><strong>Typography</strong>: I used <code>@tailwindcss/typography</code> but customized it to remove the default backticks from inline code for a cleaner look.</li>
<li><strong>Metadata</strong>: Each post automatically generates its own SEO tags and JSON-LD structured data.</li>
<li><strong>Syntax Highlighting</strong>: I chose <code>rehype-pretty-code</code> (powered by Shiki). It uses the same TextMate grammars as VS Code, so the highlighting is close to what I am used to in the editor. Because it generates inline styles, there's no brittle CSS to import from <code>node_modules</code>.</li>
<li><strong>Sitemap</strong>: A dynamic script crawls my posts directory to keep <code>sitemap.xml</code> up to date automatically.</li>
</ul>
